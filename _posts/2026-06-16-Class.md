# Java基礎知識：クラス（Class）のまとめ

## 1. クラスとは？

**クラス（Class）**とは、オブジェクトを生成するための**設計図**です。

Javaはオブジェクト指向プログラミング言語であり、プログラムはクラスを定義し、そのクラスから生成されたオブジェクトによって動作します。

> **クラスはデータ（フィールド）と処理（メソッド）をひとつにまとめ、オブジェクトを生成するための設計図です。**

* **主な特徴:**

  * データ（属性）と処理（機能）をひとつにまとめることができる。
  * オブジェクト生成の元となる設計図である。
  * コードの再利用性や保守性を高めることができる。

---

## 2. オブジェクトとは？

**オブジェクト（Object）**とは、クラスから生成された実体です。

```java
class Student {
    String name;
}

Student s = new Student();
```

* **Student** → クラス（設計図）
* **s** → オブジェクト（実体）

---

## 3. クラスの主な構成要素

### フィールド（Field）

オブジェクトが保持するデータを表します。

```java
class Student {
    String name;
    int age;
}
```

* **name**
* **age**

はフィールドと呼ばれます。

---

### メソッド（Method）

オブジェクトの振る舞いや処理を定義します。

```java
class Student {
    void study() {
        System.out.println("勉強する");
    }
}
```

**study()** がメソッドです。

---

### コンストラクタ（Constructor）

オブジェクト生成時に実行される特別なメソッドです。

```java
class Student {
    Student() {
        System.out.println("生成完了");
    }
}
```

```java
Student s = new Student();
```

オブジェクト作成時に自動的に呼び出されます。

---

## 4. インスタンスとは？

**インスタンス（Instance）**とは、クラスから生成されたオブジェクトのことです。

```java
Student s = new Student();
```

この場合、

* クラス：**Student**
* オブジェクト：**s**
* インスタンス：**s**

となります。

実務では、

> **「Studentクラスのインスタンスを生成する」**

という表現がよく使われます。

---

## 5. staticメンバとインスタンスメンバ

### インスタンスメンバ

オブジェクトを生成してから利用するメンバです。

```java
class Sample {
    int value;
}
```

```java
Sample s = new Sample();
s.value = 100;
```

---

### staticメンバ

オブジェクト生成なしで利用できるメンバです。

```java
class Sample {
    static int count;
}
```

```java
Sample.count = 10;
```

クラス名から直接アクセスできます。

---

## 6. mainメソッドにstaticが付いている理由

Javaプログラムの実行開始時には、まだオブジェクトが存在しません。

そのためJVMはオブジェクトを生成せずに呼び出せるstaticメソッドをエントリーポイントとして使用します。

```java
public static void main(String[] args)
```

> **mainメソッドはJavaプログラムの開始地点（エントリーポイント）です。**

---

## 7. クラスを使うメリット

### カプセル化（Encapsulation）

データと処理をひとつにまとめることができます。

### 再利用性（Reusability）

同じクラスから複数のオブジェクトを生成できます。

### 保守性（Maintainability）

機能ごとにクラスを分割することで管理しやすくなります。

## 9. 面接でよく深掘りされる追加質問（実務知識）

### Q. クラスとオブジェクトの違いは何ですか？

**A.** クラスは設計図であり、オブジェクトはその設計図から生成された実体です。

* **面接での補足ポイント:** 設計図がクラスで、実際に完成した家がオブジェクトであると説明すると理解しやすいです。

### Q. インスタンス変数とクラス変数の違いは何ですか？

**A.**

* **インスタンス変数** → オブジェクトごとに保持される変数

* **クラス変数（static変数）** → 全インスタンスで共有される変数

* **面接での補足ポイント:** static変数はクラスに属し、オブジェクトを生成しなくても利用できます。

### Q. コンストラクタの役割は何ですか？

**A.** オブジェクト生成時の初期化処理を行うために使用します。

* **面接での補足ポイント:** フィールドの初期値設定や生成時の共通処理を記述するために利用します。

### Q. Javaがオブジェクト指向言語と言われる理由は何ですか？

**A.** クラスとオブジェクトを中心として、カプセル化・継承・ポリモーフィズムなどのオブジェクト指向の考え方をサポートしているためです。

* **面接での補足ポイント:** Javaの三大特徴として「カプセル化・継承・ポリモーフィズム（多態性）」を説明できると評価が高くなります。

## 10. 面接用まとめ

* **クラス**：設計図
* **オブジェクト**：実体
* **インスタンス**：クラスから生成されたオブジェクト
* **フィールド**：データ
* **メソッド**：処理
* **コンストラクタ**：初期化
* **static**：クラス共通

一言で言うと、

> **「クラスはオブジェクトを生成するための設計図であり、Javaプログラムはクラスを基に生成されたオブジェクトによって動作します。」**
