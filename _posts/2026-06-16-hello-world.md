# Java基礎知識：JDK・JRE・JVMのまとめ

## 1. Javaとは？
Javaは、プログラムの作成から実行までを行うための**開発・実行プラットフォーム**です。
>  **「Write Once, Run Anywhere（一度書けば、どこでも動く）」** というスローガンの通り、OS（Windows、Mac、Linuxなど）に依存せず、同じプログラムを動かせる高いポータビリティ（移植性）が特徴です。

---

## 2. JREとは？
**JRE（Java Runtime Environment / Java実行環境）**
Javaで作成されたプログラムを**実行するためだけの環境**です。

* **主な特徴:**
  * コンパイル済みのファイル（`.class` や `.jar`）を実行できる。
  * 開発用のツールは含まれていないため、プログラムの作成やコンパイルは不可。
  * **「Javaプログラムを動かすためのパッケージ」**。
* **含まれるもの:**
  * **JVM（Java仮想マシン）**
  * **コア・ライブラリ（API）**
  * Java実行コマンド（`java` コマンド）

---

## 3. JDKとは？
**JDK（Java Development Kit / Java開発キット）**
Javaプログラムを**開発するためのツールセット（開発環境）**です。

* **主な特徴:**
  * プログラムの作成、コンパイル、デバッグ、実行のすべてが可能。
  * **「Javaプログラムを作るためのパッケージ」**。
* **含まれるもの:**
  * **JRE**（実行環境も内包）
  * **javac**（Javaコンパイラ）
  * デバッグツール（`jdb` など）
  * その他、各種開発・診断ツール

---

## 4. 各要素の関係性（構造）
それぞれの包含関係は以下の通りです。JDKが最も大きな括りとなります。

```text
[ JDK (開発環境) ]
  └── [ JRE (実行環境) ]
        ├── [ JVM (仮想マシン) ]
        └── [ 実行用ライブラリ ]
  └── [ 開発ツール (javac, デバッガ等) ]
```

* **JDK** = JRE + 開発ツール
* **JRE** = JVM + 実行ライブラリ

---

## 5. 面接対策用：一言回答集

| 質問（Q） | 回答（A） |
| :--- | :--- |
| **JREとは何ですか？** | Javaプログラムを実行するための環境です。内部にJVMや標準ライブラリを含んでいます。 |
| **JDKとは何ですか？** | Javaプログラムを開発するためのキットです。コンパイラ（`javac`）やデバッグツールのほか、実行環境であるJREも含まれています。 |
| **JREとJDKの違いは何ですか？** | JREは「実行専用」、**JDKは「開発用」**という違いがあります。JDKはJREを内包しているため、JDKをインストールすれば開発と実行の両方が可能です。 |

---

## 6. 面接でよく深掘りされる追加質問（実務・トレンド知識）

### Q. JVM（Java Virtual Machine / Java仮想マシン）とは何ですか？
**A.** Javaのクラスファイル（バイトコード）を、各OS（WindowsやLinuxなど）が理解できるマシン語に翻訳して実行する仮想マシンです。
*  **面接での補足ポイント:** JVMがOSごとの差異を吸収してくれるおかげで、開発者はOSの違いを意識せずに「Write Once, Run Anywhere（一度書けば、どこでも動く）」を実現できます。

### Q. Javaの大きな特徴である「ガベージコレクション（GC）」とは何ですか？
**A.** プログラムが動的に確保したメモリ領域のうち、使わなくなったメモリをJVMが自動で解放する機能です。
*  **面接での補足ポイント:** C++などの言語とは異なり、開発者が手動でメモリ解放（メモリ管理）を行う必要がないため、メモリリークなどのバグを防ぎやすく、安全な開発が可能です。

### Q. 【実務知識】現在のJavaにおけるJREとJDK의配布状況について知っていますか？
**A.** Java 11以降、Oracleによる「JRE単体」の提供は廃止され、JDKに統合されました。
*  **面接での補足ポイント:** 以前は「実行だけならJRE、開発ならJDK」と完全に分かれていましたが、現在のモダンなJava開発・運用現場では、サーバー環境も含めてJDKをインストールするか、必要な実行モジュールを軽量にカスタムビルドして利用するのが一般的となっています。
