# Java基礎知識：データ型（Data Type）と変数（Variable）のまとめ

## 1. データ型とは？

**データ型（Data Type）**とは、変数に格納できるデータの種類を表すものです。

Javaは**静的型付け言語**であるため、変数を宣言するときに必ずデータ型を指定する必要があります。

```java
int age = 20;
double height = 173.5;
String name = "Tanaka";
```

この場合、

* `int`
* `double`
* `String`

がデータ型です。

---

## 2. データ型の分類

Javaのデータ型は大きく以下の2種類に分類されます。

### プリミティブ型（基本データ型）

Javaが標準で提供する基本的なデータ型です。

値そのものを保持します。

### 参照型（Reference Type）

オブジェクトへの参照を保持するデータ型です。

---

## 3. プリミティブ型（基本データ型）

Javaには8種類のプリミティブ型があります。

| 分類 | 型 | サイズ | 説明 |
| :--- | :--- | :--- | :--- |
| 整数型 | `byte` | 1バイト | -128 ～ 127 |
| 整数型 | `short` | 2バイト | -32,768 ～ 32,767 |
| 整数型 | `int` | 4バイト | 一般的に使用される整数型 |
| 整数型 | `long` | 8バイト | 大きな整数を扱う |
| 浮動小数点型 | `float` | 4バイト | 約7桁の精度 |
| 浮動小数点型 | `double` | 8バイト | 約15桁の精度 |
| 文字型 | `char` | 2バイト | 1文字を扱う |
| 真偽値型 | `boolean` | JVM依存 | true または false |

---

### byte

```java
byte by = 127;
```

`byte` の範囲は

```text
-128 ～ 127
```

です。

```java
byte by = 128;
```

は範囲外のためコンパイルエラーになります。

---

### short

```java
short s = 32000;
```

比較的小さい整数を扱うための型です。

---

### int

```java
int i = 10000;
```

Javaで最もよく使用される整数型です。

特別な理由がなければ整数は `int` を使用します。

---

### long

```java
long l = 9999999999L;
```

大きな整数を扱うための型です。

`long` 型のリテラルには通常 `L` を付けます。

---

### float

```java
float f = 3.14F;
```

`float` 型のリテラルには `F` を付ける必要があります。

付けない場合は `double` として扱われます。

---

### double

```java
double d = 3.14;
```

小数を扱う際に最もよく使用される型です。

`double` の場合は `D` を省略できます。

---

### char

```java
char c = 'A';
```

1文字を格納するための型です。

```java
char c = '가';
char c2 = 'あ';
```

のように日本語や韓国語も格納できます。

> charはシングルクォート（' '）を使用します。

---

### boolean

```java
boolean flag = true;
```

真（true）または偽（false）を表す型です。

条件分岐や繰り返し処理で頻繁に使用されます。

---

## 4. 参照型（Reference Type）

参照型はオブジェクトへの参照を保持するデータ型です。

代表的な参照型：

* `String`
* 配列（Array）
* クラス（Class）
* インターフェース（Interface）

---

### String

文字列を扱うためのクラスです。

```java
String str = "Hello Java";
```

> Stringは非常によく使用されますが、プリミティブ型ではなく参照型です。

文字列はダブルクォート（" "）で囲みます。

```java
String name = "Tanaka";
```

---

### 配列（Array）

同じ型のデータを複数格納できます。

```java
int[] arr = {1, 2, 3, 4, 5};
```

配列の要素にはインデックスでアクセスします。

```java
arr[0]
arr[1]
arr[2]
```

---

## 5. 変数とは？

**変数（Variable）**とは、データを格納するための名前付きの記憶領域です。

プログラム内で値を保存したり変更したりするために使用します。

```java
int a;
```

この場合、

* データ型：`int`
* 変数名：`a`

となります。

---

## 6. 変数の宣言と初期化

### 宣言

変数を使用できる状態にすることです。

```java
int a;
```

---

### 初期化

変数へ最初の値を代入することです。

```java
a = 10;
```

---

### 宣言と初期化を同時に行う

```java
int a = 10;
```

---

## 7. 代入（Assignment）とは？

代入とは、変数へ値を格納することです。

```java
a = 10;
```

代入演算子 `=` は

```text
右側の値を左側の変数へ格納する
```

という意味です。

---

### 代入文の基本形

```java
変数 = 値;
```

また、右辺には様々なものを記述できます。

```java
int a = 10;
int b = 5;

a = b;
```

```java
a = 10 + 20;
```

```java
a = a + 1;
```

右辺には

* 値
* 変数
* 計算式
* メソッドの戻り値

などを記述できます。

---

## 8. 出力（Output）

Javaでは `System.out.println()` を使用して値を出力します。

```java
System.out.println("Hello");
```

---

### 変数の出力

```java
int a = 10;

System.out.println(a);
```

結果：

```text
10
```

---

### 文字列との連結

```java
int a = 10;

System.out.println("a = " + a);
```

結果：

```text
a = 10
```

`+` を使用すると文字列と変数を連結できます。

---

## 9. プログラム作成の基本的な流れ

初心者向けの学習では、以下の流れで考えることが多いです。

### ① 必要な変数を宣言する

```java

```

### ② 値の入力や計算を行う

```java

```

### ③ 結果を出力する

```java

```

## 11. 面接でよく深掘りされる追加質問（実務知識）

### Q. なぜStringは参照型なのですか？

**A.**
Stringはクラスとして実装されているためです。

```java
String name = "Tanaka";
```

実際にはStringオブジェクトへの参照を保持しています。

* **面接での補足ポイント:**
  Stringはよく使われますが、プリミティブ型ではありません。

### Q. intとlongの違いは何ですか？

**A.**
保持できる値の範囲が異なります。

* int → 4バイト
* long → 8バイト

longの方がより大きな整数を扱えます。

### Q. floatとdoubleの違いは何ですか？

**A.**
精度とサイズが異なります。

* float → 約7桁
* double → 約15桁

実務では通常doubleが使用されます。

### Q. charとStringの違いは何ですか？

**A.**
charは1文字、Stringは文字列を扱います。

```java
char c = 'A';
String s = "ABC";
```

* **面接での補足ポイント:**
  charはシングルクォート、Stringはダブルクォートを使用します。

## 12. 面接用まとめ

* データ型：データの種類
* プリミティブ型：値そのものを保持
* 参照型：オブジェクトへの参照を保持
* 変数：データを保存する記憶領域
* 代入：変数へ値を格納すること
* String：代表的な参照型
* 配列：複数の値を管理する仕組み
* boolean：真偽値を扱う型

一言で言うと、

> **Javaではデータ型によって扱える値が決まり、変数を利用してデータを保存・処理します。プリミティブ型は値そのものを保持し、参照型はオブジェクトへの参照を保持する点が重要です。**
