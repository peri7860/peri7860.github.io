# Java基礎知識：演算子（Operator）のまとめ

## 1. 演算子とは？

**演算子（Operator）**とは、データの計算や比較、論理的な判定を行うための記号です。

Javaでは数値計算だけでなく、条件分岐や繰り返し処理でも演算子を使用します。

```java
int a = 10;
int b = 5;

System.out.println(a + b);
```

この場合、

```java
+
```

が演算子です。

---

## 2. 演算子の種類

Javaでよく使用される演算子は以下の通りです。

| 種類 | 演算子 | 説明 |
| :--- | :--- | :--- |
| 算術演算子 | + - * / % | 四則演算 |
| 代入演算子 | = += -= *= /= %= | 値の代入 |
| 比較演算子 | == != > < >= <= | 値の比較 |
| 論理演算子 | && \|\| ! | 論理判定 |
| インクリメント・デクリメント演算子 | ++ -- | 1増加・1減少 |
| 三項演算子 | ? : | 条件による値の選択 |
| ビット演算子 | &, \|, ^, ~, <<, >> | ビット単位の演算 |

---

## 3. 算術演算子（Arithmetic Operator）

数値の計算に使用します。

| 演算子 | 説明 |
| :--- | :--- |
| + | 加算 |
| - | 減算 |
| * | 乗算 |
| / | 除算 |
| % | 剰余（余り） |

```java
int a = 10;
int b = 3;

System.out.println(a + b);
System.out.println(a - b);
System.out.println(a * b);
System.out.println(a / b);
System.out.println(a % b);
```

結果：

```text
13
7
30
3
1
```

---

### %（剰余演算子）

割り算の余りを求めます。

```java
10 % 3
```

結果：

```text
1
```

偶数・奇数判定などでよく使用されます。

```java
if(num % 2 == 0){
    System.out.println("偶数");
}
```

---

## 4. 演算子の優先順位

算術演算子には優先順位があります。

```text
① ()
② * / %
③ + -
```

同じ優先順位の場合は左から右へ計算されます。

例：

```java
System.out.println(10 + 2 * 3);
```

結果：

```text
16
```

先に

```java
2 * 3
```

が計算されます。


---

### 複合代入演算子

## 6. 比較演算子（Comparison Operator）

値を比較するための演算子です。

比較結果は必ず

```java
true
```

または

```java
false
```

になります。

| 演算子 | 説明 |
| :--- | :--- |
| == | 等しい |
| != | 等しくない |
| > | より大きい |
| < | より小さい |
| >= | 以上 |
| <= | 以下 |

例：

```java
System.out.println(10 > 5);
```

結果：

```text
true
```

## 7. 文字列比較で == を使わない理由

初心者が最も間違えやすいポイントです。

数値の場合：

```java
int a = 10;
int b = 10;

System.out.println(a == b);
```

結果：

```text
true
```

---

しかし文字列の場合は

```java
String id = "admin";

if(id == "admin"){
}
```

と書くべきではありません。

文字列の内容を比較する場合は

```java
id.equals("admin")
```

を使用します。

---

### 正しい例

```java
if(id.equals("admin")){
}
```

---

### 実務でよく見る例

```java
if(id.equals(inputId) && password.equals(inputPassword)){
}
```

* String → `equals()`
* 数値 → `==`

で比較します。

## 8. 論理演算子（Logical Operator）

複数の条件を組み合わせるために使用します。

| 演算子 | 説明 |
| :--- | :--- |
| && | AND |
| \|\| | OR |
| ! | NOT |

---

### AND（&&）

両方がtrueの場合のみtrueになります。

```java
if(age >= 20 && age <= 30){
}
```

---

### OR（||）

どちらか一方がtrueならtrueになります。

```java
if(score >= 90 || rank == 1){
}
```

---

### NOT（!）

trueとfalseを反転します。

```java
boolean flag = true;

System.out.println(!flag);
```

結果：

```text
false
```

## 11. 文字・文字列の演算

Javaでは

```java
char
```

と

```java
String
```

の動作が異なります。

---

### char + 数値

```java
System.out.println('A' + 1);
```

結果：

```text
66
```

文字コードとして計算されます。

---

### char + char

```java
System.out.println('A' + 'B');
```

結果：

```text
131
```

文字コード同士の加算になります。

---

### String + String

```java
System.out.println("A" + "B");
```

結果：

```text
AB
```

文字列連結になります。

---

### String + 数値

```java
System.out.println("A" + 1);
```

結果：

```text
A1
```

文字列として連結されます。

---

### String + String

```java
System.out.println("1" + "2");
```

結果：

```text
12
```

数値計算ではなく文字列連結です。

---

### 数値と文字列を組み合わせた演算

```java
System.out.println(1 + 2 + "3");
```

結果：

```text
33
```

数値同士の計算が先に行われるため、

```java
1 + 2
```

が先に計算されて `3` となり、その後文字列 `"3"` と連結されます。

---

```java
System.out.println("1" + 2 + 3);
```

結果：

```text
123
```

文字列との連結が始まると、それ以降はすべて文字列として扱われます。

まず

```java
"1" + 2
```

が `"12"` となり、その後

```java
"12" + 3
```

が実行されるため、結果は `"123"` になります。

---

### 補足

`+` 演算子は、文字列が1つでも含まれると文字列連結として動作します。

そのため、演算の順序によって結果が変わります。

```java
System.out.println(1 + 2 + "3");   // 33
System.out.println("1" + 2 + 3);   // 123
```

実務でもログ出力やメッセージ作成時によく使用されるため、計算が先に行われるのか、文字列連結が先に行われるのかを理解しておくことが重要です。

## 15. 面接でよく深掘りされる追加質問（実務知識・より深い回答）

### Q. Stringの比較で `==` ではなく `equals()` を使う理由は何ですか？

**A.**

`==` はオブジェクトの参照先（メモリアドレス）を比較するのに対し、`equals()` は文字列の内容そのものを比較するためです。

```java
String a = "Java";
String b = "Java";

System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true
```

しかし、

```java
String a = new String("Java");
String b = new String("Java");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
```

のように、文字列の内容が同じでも参照先が異なる場合があります。

* **面接での補足ポイント**
  Javaには **String Pool（文字列プール）** という仕組みがあります。同じ文字列リテラルは同じ参照を共有するため、偶然 `==` が `true` になることがあります。しかし、`new String()` などで生成した場合は参照が異なるため、実務では文字列比較に `equals()` を使用するのが一般的です。Java 7以降では Objects.equals() を利用する方法もあります。

さらに、NullPointerExceptionを防ぐために、

```java
if("admin".equals(id)){
}
```

のように、定数側から `equals()` を呼び出す書き方が推奨されます。

```java
// 悪い例
if(id.equals("admin")){
}

// 良い例
if("admin".equals(id)){
}
```

---

### Q. `&&` と `&` の違いは何ですか？

**A.**

`&&` は論理演算子であり、**短絡評価（ショートサーキット）** を行います。

```java
if(a > 0 && b > 0){
}
```

左側が `false` の場合、右側は評価されません。

一方、

```java
&
```

はビット演算子であり、条件式で使用した場合でも左右を必ず評価します。

```java
if(a > 0 & b > 0){
}
```

こちらは左側が `false` でも右側を評価します。

* **面接での補足ポイント**
  実務では短絡評価を利用してエラーを防ぐことがよくあります。

```java
if(str != null && str.length() > 0){
}
```

このコードでは、`str` が `null` の場合に右側の `str.length()` が実行されないため、`NullPointerException` を回避できます。

---

### Q. 三項演算子とif文の違いは何ですか？

**A.**

どちらも条件分岐を行いますが、三項演算子は **値を返す式（Expression）** であり、if文は **処理を分岐させる文（Statement）** です。

```java
String result =
    score >= 60 ? "合格" : "不合格";
```

三項演算子は条件によって1つの値を返します。

一方、if文は処理そのものを分岐させます。

```java
if(score >= 60){
    System.out.println("合格");
}else{
    System.out.println("不合格");
}
```

* **面接での補足ポイント**
  単純な値の代入であれば三項演算子を使用するとコードを簡潔に書けます。

```java
String status =
    score >= 80 ? "優秀" : "通常";
```

ただし、三項演算子をネストすると可読性が大きく低下します。

```java
// あまり推奨されない例
String grade =
    score >= 90 ? "A" :
    score >= 80 ? "B" :
    score >= 70 ? "C" : "D";
```

条件が複雑になる場合や処理を伴う場合は、if文を使用するのが一般的です。

## 16. 面接用まとめ

* 演算子：計算や比較を行う記号
* 算術演算子：+ - * / %
* 比較演算子：== != > < >= <=
* 論理演算子：&& || !
* インクリメント：++
* デクリメント：--
* 三項演算子：条件 ? 真 : 偽
* String比較：equals()
* ビット演算子：&, |, ^, ~, <<, >>

一言で言うと、

> **演算子とは、データの計算・比較・条件判定を行うための記号です。特に実務では比較演算子、論理演算子、Stringのequals()が頻繁に使用されます。**
